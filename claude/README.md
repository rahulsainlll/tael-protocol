# Tael — Project Context (`claude/`)

This folder holds the **product goals, roadmap, and principles** for Tael — the durable _what_ and
_why_ of the project. It's the north star we check work against.

For the _how it's built_ (folder structure, package boundaries, tooling, conventions), see
[`../ARCHITECTURE.md`](../ARCHITECTURE.md).

| File                               | Purpose                                                             |
| ---------------------------------- | ------------------------------------------------------------------- |
| [`goals.md`](./goals.md)           | Mission, the problem, the solution, target users, success criteria  |
| [`roadmap.md`](./roadmap.md)       | Phased delivery plan and current status                             |
| [`product.md`](./product.md)       | Product spec — dashboard, marketplace, capability model, user flows |
| [`principles.md`](./principles.md) | Engineering principles and the quality bar                          |

> Keep these updated as the product direction evolves. Code and structure change often; this folder
> should change only when the _goals_ do.
